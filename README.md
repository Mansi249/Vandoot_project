**VanDoot** is an integrated IoT ecosystem designed for real-time forest threat detection. It solves the problem of "False Alarms" in remote monitoring by using a **Multi-Sensor Fusion** strategy—fusing computer vision with environmental data to provide 100% reliable alerts.

---

## 🛠️ The VanDoot Pipeline (How it Works)

This repository is organized according to the data flow of the system. Here is the step-by-step breakdown:

### 1️⃣ Stage 1: Edge Vision (The Sentry)
The **`esp32/`** directory contains the "eyes" of the system.
* **Firmware**: `esp32_ai_camera.ino` runs on an ESP32-CAM.
* **On-Device AI**: It uses a quantized **MobileNetV1** model (`model_data.cc`) to classify images into Fire, Human, or Animal.
* **Logic**: Only significant detections are logged to save power and bandwidth.

### 2️⃣ Stage 2: Sensor Fusion (The Judge)
The **`random_forest/`** directory contains the "brain" that prevents false positives.
* **The Problem**: A camera might see a "red sunset" and think it is a "Fire."
* **The Solution**: The `vandoot_judge.pkl` (Random Forest model) fuses vision logs with Smoke, Heat, and Audio data.
* **Decision Logic**: 
    * `Vision(Fire)` + `Smoke(Low)` = **SAFE** (System overrides the camera).
    * `Vision(Human)` + `Audio(High)` = **POACHER** (Detected chainsaw/logging noise).

### 3️⃣ Stage 3: Analytical Dashboard (Intelligence)
The **`streamlit_app.py`** is the final interface for forest rangers.
* **Data Processing**: It reads the `logs/events.txt` generated by the sensors.
* **Generative AI**: It integrates with **Gemini AI** to transform raw event logs into "Ecological Intelligence Reports," summarizing long-term threat patterns in plain English.

## 📂 Repository Map
* `data_generation/`: Contains `generate_realistic_logs.py` to simulate 180 days of forest activity for testing.
* `esp32/`: Contains the MobileNetV1 architecture and Arduino deployment code.
* `random_forest/`: Contains the training scripts (`train_rf.py`) for the fusion logic.
* `logs/`: The data bridge where sensors record their findings.

## 🚀 Quick Start (Single-Terminal Pipeline)

To simulate the entire system and launch the dashboard:

```bash
# 1. Install necessary libraries
pip install streamlit google-generativeai python-dotenv

# 2. Add your Gemini API Key
export GOOGLE_API_KEY='your_api_key_here'

# 3. Generate logs and launch the app
python data_generation/generate_realistic_logs.py && streamlit run streamlit_app.py
